Runtime safety for Postgres writes

Permissions say who. Interdict asks how much, and can we undo it?

Interdict sits between whoever is writing SQL, an AI coding agent or a human, and Postgres. It stops dangerous writes before they happen by previewing real blast radius, requiring confirmation, returning structured fixes, and recording reversible writes.

The gap

Roles and grants do not prevent the real database accidents.

Postgres permissions can answer whether a role may touch a table. They cannot answer whether this exact statement changes one row or 2,300,000 rows. They cannot tell you whether the write can be reversed without restoring an entire backup.

Interdict adds a deterministic runtime gate for that missing layer: statement shape, policy, blast radius, confirmation, audit, and undo before a bad agent or human query becomes production damage.

How it works

Every statement gets the same deterministic decision path.

01

Parse

Use a real Postgres parser, not regex, so comments, casing, aliases, and footguns like stray semicolons are handled structurally.

02

Classify

Identify reads, writes, DDL, catalog access, risky functions, broad scopes, and unsupported statement shapes.

03

Simulate

For risky writes, run a time-boxed throwaway transaction and roll it back to measure the real affected-row count.

04

Decide

Allow, block, hold for confirmation, audit, and record an undo path when the write can be safely reversed.

Blast-radius preview

Impact is measured against the database, not guessed from SQL text.

Instant undo

Reverse one bad action by id instead of restoring a whole dump and losing everyone else's work.

Structured rejection

Humans get plain English. Agents get reason codes and suggested fixes they can retry against.

Fast hot path

Deterministic checks run in microseconds; the end-to-end benchmark reports ~0 ms overhead.

Human Mode

You type SQL at the prompt.

Reads and scoped writes run. Risky writes show a blast-radius confirm panel. \undo and \revert <id> restore specific writes. \override is audited.

Agent Mode

Claude Code or Codex calls a guarded tool.

Add the MCP server once. The model calls run_query instead of touching Postgres directly. Approval uses an operator token the model never sees.
